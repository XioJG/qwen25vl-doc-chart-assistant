# Screenshots

本目录用于放置可以公开发布的脱敏截图。截图应服务于项目说明文档，而不是堆放所有调试图片。

## 推荐截图类型

1. Gradio Demo 总览截图

   建议命名：

   ```text
   gradio_demo_overview.png
   ```

   内容建议：

   - 左侧显示上传的脱敏文档/图表/界面截图。
   - 中间或左侧显示任务模板和问题。
   - 右侧显示模型原始回答和解析出的 JSON。

   作用：说明项目有本地 Demo，不只是脚本。

2. 命令行推理与输出截图

   建议命名：

   ```text
   cli_single_image_inference.png
   ```

   内容建议：

   - 包含 `python -m src.inference_cli ...` 的运行命令。
   - 包含 prompt、模型回答和保存路径。

   作用：说明项目可以脱离界面，通过 CLI 复现实验和调试。

3. 结构化 JSON 抽取效果截图

   建议命名：

   ```text
   structured_json_output.png
   ```

   内容建议：

   - 包含一个脱敏输入截图。
   - 包含模型输出的 JSON 字段。
   - 字段可以是示例票据、表格、图表或 UI 页面中的非敏感信息。

   作用：说明“文档/图表理解 + 信息抽取 + 可机器读取输出”的处理目标。

## 截图要求

- 不要出现真实姓名、手机号、地址、邮箱、账号、订单号、token、公司内部系统或敏感业务字段。
- 不要出现本地绝对路径，例如 `/home/用户名/...`。
- 建议使用公开样例、合成数据或完全脱敏后的截图。
- 图片尺寸建议保持清晰可读，优先使用 PNG。
